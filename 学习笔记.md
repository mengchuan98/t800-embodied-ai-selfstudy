## 第 01 课 · 具身智能仿真概念

**核心概念**
- **具身智能（Embodied AI）**：让 AI 拥有"身体"（机器人），通过与环境交互来感知、决策、行动，而不只是处理纯文本/图像。
- 本课程用 **众擎（EngineAI）T800** 人形机器人作为载体，在 **MuJoCo** 物理仿真器里做开发。
- **为什么要仿真**：真实机器人贵、易损坏、调试慢；先在仿真里把控制策略（"大脑"）训练/验证好，再迁移到真机。

**学习路径（本课到 13 课的全景）**
1. 认识仿真器（MuJoCo Hello World）
2. 加载 T800 模型
3. 理解关节(joint)、连杆(link)、坐标
4. 理解状态参数（nq/nv/nu、qpos/qvel/ctrl）
5. 仿真主循环
6. 控制：PD 控制 → 力矩上限 → 手动控制 → AI 策略网络
7. 观测空间：把机器人状态加工成策略网络能用的输入（obs）
8. 部署闭环：AI 指令(action) → default_Q + action_scale → 肌肉(ctrl) → 行走

**关键工具**
- `mujoco`：物理仿真核心库
- `mujoco.viewer`：自带 3D 可视化窗口
- 课程 conda 环境：`t800`

## 第 02 课 · MuJoCo Hello World（让方块掉下去）

**目标**：跑通第一个 MuJoCo 程序——一个木块在重力下自由落体，并定时打印高度。

**对应代码**：`01_mujoco_helloworld.py`

**核心知识点**
- 用 **XML 字符串** 描述场景（也可以用 `.xml` 文件，`from_xml_path`）：
  - `<option gravity="0 0 -9.81" timestep="0.002"/>`：重力沿 -Z（9.81 m/s²），仿真步长 0.002s（即 500 Hz）。
  - `<geom type="plane" .../>`：无限大地面。
  - `<body name="block">` + `<freejoint/>`：自由体（6 自由度，可平移+旋转）。
  - `<geom type="box" size="0.1 0.1 0.1" mass="1"/>`：1kg 的 0.2m 立方体。
- 加载与仿真：
  ```python
  model = mujoco.MjModel.from_xml_string(XML)
  data  = mujoco.MjData(model)
  while data.time < SIM_DURATION:
      mujoco.mj_step(model, data)        # 推进一帧物理仿真
  ```
- `freejoint` 的 `qpos` 前 3 个分量就是 body 的位置 (x, y, z)：
  ```python
  height = data.qpos[2]   # 取 z 分量即高度
  ```
- 实时同步：用 `time.sleep` 让"1 秒仿真 ≈ 1 秒真实时间"，避免循环跑太快。

**要点**
- **MjModel**：模型的"静态定义"（结构、质量、关节、电机等都不变）。
- **MjData**：仿真的"动态状态"（每一帧的位置、速度、控制量都在这儿）。
- 一次典型流程：`from_xml_*` 得到 model → 创建 data → `mj_step` 循环推进。

## 第 03 课 · 加载 T800（银色机器人第一次亮相）

**目标**：加载课程提供的 `t800.xml`，认识模型整体规模，并把机器人显示出来。

**对应代码**：`02_t800_info.py`、`04_t800_viewer.py`

**核心知识点**
- 从文件加载（`<include>` 的相对路径会基于 xml 文件所在目录解析）：

  XML_PATH = os.path.join(os.path.dirname(__file__), "t800.xml")
  model = mujoco.MjModel.from_xml_path(XML_PATH)
  ```
- 模型规模三件套（T800 实测值）：

  model.nbody   # 身体/连杆部件总数（含 worldbody，worldbody 通常是 id=0）= 31

- 遍历名字核对结构：

  for i in range(model.nbody):  print(model.body(i).name)    # 连杆

  for i in range(model.nu):     print(model.actuator(i).name)# 电机

          viewer.sync()                      # 把最新状态同步到画面
          time.sleep(model.opt.timestep)
  ```

**要点**
- T800 是一个双足人形机器人，`t800.xml` 已经把躯干、髋、膝、脚、腰、肩、肘、腕、头等连杆和对应关节、电机都定义好了。

## 第 04 课 · 认识 T800 的关节(joint)和连杆(link)（控制的花名册）

**目标**：搞清楚 T800 有哪些关节、哪些连杆，以及它们的类型——这是后续"控制谁"的基础。

**对应代码**：`05_t800_print_all_joints.py`、`06_t800_print_all_links.py`

**核心知识点（关节 joint）**
- 关节类型枚举（代码里的映射）：
  | 枚举值 | 类型 | 说明 |
  |---|---|---|
  | 0 | `mjJNT_FREE` | 自由关节（6 自由度，本体的 base 用） |
  | 1 | `mjJNT_BALL` | 球关节（3 自由度） |
  | 2 | `mjJNT_SLIDE` | 滑动关节（1 自由度，平移） |
  | 3 | `mjJNT_HINGE` | 旋转关节（1 自由度，最常用，如髋/膝/肩/肘） |
- 每个关节的关键地址（用来在 `qpos`/`qvel` 数组里定位）：
  ```python
  model.jnt_type[i]      # 关节类型
  model.jnt_qposadr[i]   # 在 qpos 数组中的起始下标
  model.jnt_dofadr[i]    # 在速度空间(dof)中的起始下标
  ```
- 示例打印格式：`[000] <unnamed_0> | type=mjJNT_FREE | qposadr=0 | dofadr=0`

**核心知识点（连杆 link / body）**
- 连杆是"刚体部件"，关节连接父子连杆，形成运动链。
- 每个 body 可带多个几何体(geom)，几何体有类型与尺寸：
  | geom 类型 | 含义 |
  |---|---|
  | 0 | plane 地面 |
  | 1 | sphere 球 |
  | 2 | capsule 胶囊 |
  | 5 | box 方块 |
  | 6 | mesh 网格（机器人常用，外观精细） |
- 读取连杆信息：
  ```python
  model.body_mass[i]        # 质量
  model.body_geomnum[i]     # 几何体数量
  model.body_geomadr[i]     # 第一个几何体下标
  model.geom_type[g]        # 几何体类型
  model.geom_size[g]        # 几何体尺寸（三元组）
  ```
- 注意 `model.nbody` 包含 worldbody（id=0），真实连杆要从 `1` 开始遍历。

## 第 05 课 · T800 相对坐标和绝对坐标（关节的位置关系）

**目标**：理解"相对坐标"和"绝对坐标"，并学会把机器人骨架画出来。

**对应代码**：`07_t800_print_all_body_pos.py`、`08_t800_print_all_body_xpos.py`、`09_t800_skeleton_plot.py`、`09_01_t800_skeleton_views.py`

**核心知识点**
- MuJoCo 里每个 body 在 XML 中写的位置是 **相对父连杆的局部坐标**（relative/local）。
- 仿真运行后，能得到每个 body 的 **世界坐标（绝对坐标）**：
  ```python
  mujoco.mj_forward(model, data)   # 先前向计算一次，让 xpos 等派生量有效
  data.xpos[i]                     # body i 的世界坐标 (x, y, z)
  data.xquat[i]                    # body i 的世界朝向（四元数 [w,x,y,z]）
  ```
- `07 / 08` 两个脚本本质都是打印 `data.xpos`（`07` 用 `body_pose` 字段名，`08` 直接叫 `xpos`，效果一致）。
- **骨架可视化**：用 `body_parentid` 自动构建父子连线，再用 matplotlib 3D 画出机器人骨架：
  ```python
  for i in range(model.nbody):
      parent = int(model.body_parentid[i])   # 父连杆 id
      # 把 (parent -> i) 连成一条边
  ```
  - `09_t800_skeleton_plot.py`：单图带中文标签（躯干/左髋/左膝/左脚…）。
  - `09_01_t800_skeleton_views.py`：正视图/侧视图/俯视图三视图。
  - 关键中文标签映射示例：`LINK_BASE→躯干`、`LINK_HIP_PITCH_L→左髋`、`LINK_KNEE_PITCH_L→左膝`、`LINK_FOOT_L→左脚`、`LINK_HEAD_YAW→头`。

**要点**
- "相对坐标"用于建模（在 xml 里写关节相对父级的位置）；"绝对坐标"(`xpos`)用于运行时读取真实世界位姿。两者通过正运动学（forward kinematics）相互转换，调用 `mj_forward` 即可刷新。

## 第 06 课 · T800 状态参数（nq / nv / nu / qpos / qvel / ctrl）

**目标**：掌握 MuJoCo 描述"机器人当前状态"的几个核心向量，这是写任何控制器都要打交道的数据。

**对应代码**：`03_t800_nq_nv_nu.py`、`10_t800_print_state_info.py`

**核心概念对照表**
| 名称 | 含义 | 长度 | 存哪里 |
|---|---|---|---|
| `nq` | 广义坐标数量（位形空间维度） | = qpos 长度 | `model.nq` |
| `nv` | 广义速度数量（速度空间维度） | = qvel 长度 | `model.nv` |
| `nu` | 执行器/电机数量（控制维度） | = ctrl 长度 | `model.nu` |
| `qpos` | 关节位置（角度/位移） | nq | `data.qpos` |
| `qvel` | 关节速度 | nv | `data.qvel` |
| `ctrl` | 控制量（发给电机的指令，常是目标力矩） | nu | `data.ctrl` |

```python
print(model.nq, model.nv, model.nu)   # T800 实测: nq=32, nv=31, nu=25
print(data.qpos)   # 初始关节角度（弧度）
print(data.qvel)   # 初始速度（通常全 0）
print(data.ctrl)   # 初始控制量（通常全 0）
```

**要点**
- T800 实测：**nq=32、nv=31、nu=25**。
- 维度构成：base 的 free joint 贡献 7 维 qpos（3 位置 + 4 四元数）、6 维 qvel（3 线速度 + 3 角速度）；其余 25 个可驱动 HINGE 关节各贡献 1 维。所以 `7+25=32=nq`、`6+25=31=nv`，`nu=25` 即 25 个电机。
- `nu` 等于可控关节（带电机的 HINGE）数量 25，注意它**不等于**后面第 11 课 MNN 策略输出的 22 维动作（见第 11 课说明）。
- 写控制器时：读 `qpos/qvel` → 算 `ctrl` → 写回 `data.ctrl` → `mj_step`。

## 第 07 课 · MuJoCo 仿真主循环（机器人世界的一帧）

**目标**：理解仿真是如何"一帧一帧"推进的，以及如何在循环里读取位姿。

**对应代码**：`04_t800_viewer.py`、`11_t800_viewer_print_pose_rpy.py`

**主循环范式**
```python
while viewer.is_running() and data.time < DURATION:
    mujoco.mj_step(model, data)   # 1) 物理推进一帧（根据 ctrl、力、重力算新状态）
    viewer.sync()                 # 2) 刷新 3D 画面
    time.sleep(model.opt.timestep)  # 3) 实时节流
```

**读取躯干(base)位姿与欧拉角**
- T800 的躯干 body 名为 `LINK_BASE`：
  ```python
  base_id = model.body("LINK_BASE").id
  pos   = data.xpos[base_id]     # 世界坐标 (x前后, y左右, z高度)
  quat  = data.xquat[base_id]    # 世界朝向四元数 [w,x,y,z]
  ```
- 四元数 → 欧拉角（roll/pitch/yaw）：
  ```python
  def quat_to_rpy(q):
      w, x, y, z = q
      roll  = math.atan2(2*(w*x + y*z), 1 - 2*(x*x + y*y))   # 绕 X
      pitch = math.asin(max(-1, min(1, 2*(w*y - z*x))))      # 绕 Y
      yaw   = math.atan2(2*(w*z + x*y), 1 - 2*(y*y + z*z))   # 绕 Z
      return roll, pitch, yaw
  ```
- 坐标约定（代码注释）：pos 中 X=前后、Y=左右、Z=高度（单位 m）；rpy 单位 rad。

**要点**
- `mj_step` 是"世界前进一帧"的核心。你在 step 之前写入 `data.ctrl`，step 之后读取 `data.qpos/qvel/xpos` 等。
- 周期性（如每 100ms）打印位姿，用来观察机器人是否稳定、有没有漂移。

## 第 08 课 · PD 控制（给 T800 上发条）

**目标**：用最经典的 PD 控制器，让每个关节"稳住/回到"目标角度——这是机器人站立、动作的基础控制律。

**对应代码**：`12_t800_pd_hold_init_qpos.py`

**PD 控制律**
```
u = kp * (q_des - q) - kd * qdot
```
- `q_des`：目标关节角度（这里取初始 qpos）。
- `q`：当前关节角度；`qdot`：当前关节速度。
- `kp`：比例增益（拉回目标）；`kd`：微分增益（阻尼，抑制抖动）。
- 经验取法：`kd = 2 * sqrt(kp)`（临界阻尼附近），示例用 `kp=5.0`。

**实现要点**
```python
qpos0 = data.qpos.copy()               # 记录初始目标
for aid in range(model.nu):
    jnt_id = int(model.actuator_trnid[aid, 0])   # 电机关联的关节
    if jnt_type not in (HINGE, SLIDE):           # 跳过 FREE/BALL 多自由度
        continue
    qposadr = int(model.jnt_qposadr[jnt_id])
    dofadr  = int(model.jnt_dofadr[jnt_id])
    q, qd   = data.qpos[qposadr], data.qvel[dofadr]
    u = kp * (qpos0[qposadr] - q) - kd * qd
    # 限幅（见第 09 课）
    data.ctrl[aid] = u
mujoco.mj_step(model, data)
```
- 通过 `model.actuator_trnid[aid, 0]` 找到"电机 → 关节"的映射，这是写控制器时最常查的字段。
- 只对 HINGE/SLIDE（单自由度）做 PD；FREE/BALL 多自由度关节不能简单套用。

**要点**
- PD 是高高频、低层的"肌肉反射"；真正让机器人走、跑的是更上层（第 11 课）的策略网络，但底层仍靠 PD 把策略输出的目标转成力矩。

## 第 09 课 · 力矩天花板（control range）

**目标**：理解每个电机都有输出力矩上限，写控制器时必须限幅，否则会出现"命令力矩无穷大"的荒谬情况。

**对应代码**：`13_t800_print_actuator_ctrlrange.py`、`14_t800_test_clamp_ctrl.py`

**核心知识点**
- 每个 actuator 在 xml 里定义了控制范围 `ctrlrange`：
  ```python
  model.actuator_ctrllimited[aid]   # 是否有限幅（0/1）
  model.actuator_ctrlrange[aid]     # [lo, hi] 控制量上下限
  model.actuator_trnid[aid, 0]      # 关联的关节 id（用来看名字）
  ```
- 打印每个电机的可控范围（含对应关节名）：
  ```python
  name   = mujoco.mj_id2name(model, mujoco.mjtObj.mjOBJ_ACTUATOR, aid)
  jnt_id = int(model.actuator_trnid[aid, 0])
  jname  = mujoco.mj_id2name(model, mujoco.mjtObj.mjOBJ_JOINT, jnt_id)
  ```
- **限幅函数（clamp）**：PD 算出的 `u` 必须夹在 `[lo, hi]`：
  ```python
  def clamp(x, lo, hi):
      return lo if x < lo else hi if x > hi else x
  # 应用：
  if int(model.actuator_ctrllimited[aid]):
      u = clamp(u, ctrlrange[0], ctrlrange[1])
  ```
- `14` 号脚本做了极端测试：把 ctrl 设成 ±10000，再 clamp，验证"真实值"被夹回 `ctrlrange` 内，说明"力矩天花板"生效。

**要点**
- 真实电机力矩有限，仿真里也必须尊重 `ctrlrange`，否则物理会发散/爆掉。
- 写任何控制器，最后一步几乎都是 `ctrl = clamp(ctrl, lo, hi)`。

## 第 10 课 · 手动控制 T800 机器人（mujoco gui）

**目标**：用 GUI 拖滑条手动控制关节，直观感受每个关节的作用——调试与验证模型的好办法。

**对应代码**：`15_t800_manual_control.py`、`16_t800_native_control.py`

**两种方式**
1. **MuJoCo 原生查看器（16 号）**：最省事，直接打开 `launch_passive` 后，在窗口右侧 **"Control" 标签页** 拖动电机滑条即可施加扭矩。
   ```python
   with mujoco.viewer.launch_passive(model, data) as viewer:
       while viewer.is_running():
           mujoco.mj_step(model, data); viewer.sync()
           time.sleep(model.opt.timestep)
   ```
2. **自定义 tkinter 面板（15 号）**：教学演示版，滑条直接改 `qpos`（瞬移关节角度），**不走 PD、不施加力矩**，所以最直观稳定：
   - 收集所有 HINGE/SLIDE 关节 → 每个滑条对应一个 `qposadr`，范围用 `jnt_range`（转成角度 deg 显示）。
   - 固定 base 的 free joint（`data.qpos[:7]` 保持初始），防止机器人一上来就倒。
   - 每帧把滑条角度(deg)转弧度写进 `data.qpos`，再 `mj_forward` 刷新正运动学。

**要点**
- 手动控制常用于：确认关节方向对不对、限位合不合理、模型是否装配正常。
- "改 qpos"适合静态摆 pose；"改 ctrl"才是动力学驱动（配合 PD/策略）。

## 第 11 课 · AI 策略生成关节运控指令（MNN 策略网络给机器人接上大脑）

**目标**：把训练好的神经网络策略接到机器人上，让它根据"观测(observation)"自动输出"动作(action)"，取代手写规则。

**对应代码**：`17_t800_mnn_action_test.py`

**核心知识点**
- 用 **MNN**（轻量推理引擎）加载训练好的策略模型：
  ```python
  import MNN
  net = MNN.Interpreter("policy/t800_260318_150533_60000.mnn")
  session = net.createSession()
  input_tensor  = net.getSessionInput(session)
  output_tensor = net.getSessionOutput(session)
  ```
- 推理（前向传播）：
  ```python
  obs = np.zeros(OBS_SIZE, dtype=np.float32)   # 1083 维观测
  # 把 obs 拷进 input tensor → runSession → 从 output tensor 拷出 action
  action = bridge.infer(obs)                    # 22 维动作
  ```
- **维度约定（本课关键数字）**：
  - `OBS_SIZE = 1083`：观测向量长度，具体构成见第 12 课（`= 15 帧历史本体感知 × 72 维/帧 + 3 维命令`）。
  - `ACTION_SIZE = 22`：策略网络输出的动作向量长度 = **22**。
  - ⚠️ **注意差异**：当前 `t800.xml` 实测 `nu = 25`（25 个电机），而策略只输出 **22** 维动作，**两者不相等**。对应关系（以 `18_t800_mnn_walk.py` 的映射为准）：`action[i]` 直接驱动第 `i` 个电机（`aid = 0..21`，即 `J00`–`J21`：双腿 12 个 + 腰 `TORSO_YAW` + 双臂 + 左右 `ELBOW_PITCH` 等），而 `aid = 22..24`（`J22_ELBOW_YAW_R`、`J23_HEAD_PITCH`、`J24_HEAD_YAW`）保持默认姿态、由 PD 维持站立。即策略覆盖 22 个电机，剩余 3 个由站立 PD 保持。把 `action` 映射到 `ctrl` 时要确认对齐方式（代码 17 仅做推理测试，未做映射）。具体以课程后续讲到的"动作→关节"映射为准。
- 推理后，这 22 维 action 通常会作为各关节 **PD 控制器的目标 `q_des`**（接回第 08 课的 PD 框架），从而形成"策略网络(大脑) → PD(肌肉) → 机器人"的完整控制链。

**要点**
- 这一课是"接大脑"：前面 02–10 课搭好了身体（模型）、感知（状态/坐标）、肌肉（PD+限幅）、手脚（手动控制）；本课把训练好的神经网络策略接上去，机器人就能"自己动"。
- 推理时务必核对维度：`obs` 必须是 1083、`action` 必须是 22，否则说明观测拼装或模型不匹配。

## 第 12 课 · T800 的观测空间（observation space）

**目标**：搞清楚"喂给 AI 策略网络的输入长什么样"——即观测空间（observation space）的维度 `1083` 是怎么来的。

**核心概念**
- **观测空间（observation space）**：策略网络每一步"看到"的信息集合，是网络真正的输入。机器人不能"看"世界，只能看这一串数字——所以观测怎么设计，直接决定机器人能感知什么、能学会什么。
- **总维度 1083 的来源**（PPT 标题即给出口诀）：
  ```
  1083 = 72 × 15 + 3
  ```
  - **`72`**：每一"帧"本体感知的特征维度（一个时刻机器人的自身状态被编码成 72 个数）。
  - **`15`**：把最近 **15 个历史时刻** 的本体感知**按时间堆叠**（stack），让网络能感知"变化趋势/动态"，而不只是瞬时快照。
  - **`3`**：额外拼接的 **命令/目标向量**（通常 3 维，例如期望的行走速度 `vx, vy` 与转向角速度 `vyaw`），告诉机器人"往哪走、走多快"。

**这 72 维/帧通常包含什么（本体感知 proprioception）**
- 各**可驱动关节的角度（`qpos`，25 维）** 与 **角速度（`qvel`，25 维）**；
- **base 的位姿与速度**（IMU 信息，如朝向/重力投影、角速度、线速度等）；
- 可能的**脚部/接触信息**、足端相对位置等。
- 这些拼起来每帧约 72 维；连续 15 帧 + 3 维命令 = 1083。

**为什么需要"历史帧"**
- 单帧只有瞬时状态，网络无法判断"正在加速还是减速、脚是否刚着地"。堆叠 15 帧相当于给网络一点"短期记忆"，才能做出平稳的行走决策。

**与前后课的关系（闭环）**
- 第 06 课：机器人状态向量 `qpos / qvel / ctrl`（原始来源）。
- 本课：把状态加工成 **观测 obs（1083 维）**——叠加历史 + 命令。
- 第 11 课：MNN 策略 `obs(1083) → action(22)`，22 维动作再经 PD（第 08 课）+ 限幅（第 09 课）变成 `ctrl` 驱动机器人。

**要点**
- 观测空间 ≠ 控制空间：obs 是"输入/我看到什么"（1083），action 是"输出/我做什么"（22），二者维度本就可不同。
- `1083` 与 `22` 都是**训练这个策略时固定下来的约定**，推理时必须严格对齐；任意一维错位，机器人行为都会异常。
- 设计观测的常见原则：用**本体感知**（关节、IMU）而非原始像素，工程简单、泛化好；命令向量让同一个网络能执行不同速度/方向。

## 第 13 课 · AI 指令到肌肉运动（打通任督二脉）

**目标**：把 AI 策略输出的 `action` 真正"接到"机器人肌肉上——把 22 维 action 变成能驱动关节的 `ctrl`，完成 `obs → action → ctrl` 闭环，机器人从此能自己行走。

**两个关键概念（PPT 关键词）**
- **`default_Q`（默认姿态）**：机器人站立时的初始关节角度，是策略动作的"基准姿态"。策略输出的 `action` 通常是**相对 `default_Q` 的偏移量**，而非绝对角度——这样机器人动作始终以"站立"为起点，不会突然跳到怪异姿态。
- **`action_scale`（动作缩放）**：把 `action` 从"策略输出尺度"放大到"真实关节角度偏移尺度"的增益系数。策略输出往往数值很小或已归一化，必须乘上 `action_scale` 才对应真实弧度。

**完整控制链路（打通任督二脉）**
```
obs (1083 维)                          ← 第 12 课：状态加工成观测
   │  MNN 推理（第 11 课）
   ▼
action (22 维)                         ← 策略输出的动作
   │  q_des = default_Q + action_scale × action   ← 本课本职
   │  PD 控制（第 08 课）：u = kp·(q_des − q) − kd·q̇
   │  力矩限幅（第 09 课）：clamp 到 ctrlrange
   ▼
ctrl (25 维)                           ← 写 data.ctrl
   │  mj_step
   ▼
机器人向前行走
```

**对应代码**：`18_t800_mnn_walk.py`
- `build_obs(model, data, base_id)`：每步把状态拼成 1083 维观测（第 12 课）。
- `policy.act(obs)`：MNN 推理得到 22 维 `action`（第 11 课）。
- 主循环：`q_des = qpos0[adr] + action[aid]`（`qpos0` 即 `default_Q`；教学版暂未乘 `action_scale`，若课程给出缩放系数需补上）→ PD 算 `u` → `ctrlrange` 限幅 → 写入 `data.ctrl[aid]` → `mj_step`。
- `qpos0` 来自仿真启动瞬间的 `data.qpos.copy()`，也就是 T800 的站立初始姿态。

**要点**
- 这一课把"大脑"(第 11 课)和"肌肉"(第 8/9 课)真正连起来：`action` 不再是测试数据，而是每步实时驱动机器人。
- `default_Q` 与 `action_scale` 都是**训练策略时一起确定的约定**，部署时必须和训练端一致，否则动作幅度/基准会错。
- 第 11 课提到的 `nu=25` 与 `action=22` 差异在此落地：22 维 `action` 对应 22 个被策略驱动的关节，其余关节直接保持 `default_Q`（由 PD 维持站立）。
